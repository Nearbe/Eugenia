# Product Guide: Eugenia

## 1.0 Product Vision
Eugenia — это система двойного назначения, объединяющая высокоточную топологическую визуализацию данных и детерминированную систему извлечения знаний из нейронных сетей. Проект трансформирует хаотичные веса моделей в структурируемые, интерпретируемые паттерны.

## 2.0 Target Users
- Исследователи в области интерпретируемости ИИ (AI interpretability)
- Data scientists, работающие с анализом визуальных данных
- Разработчики, изучающие топологический анализ изображений
- Математики, интересующиеся дельта-полями и топологическими инвариантами

## 3.0 Core Goals
1. **Delta Field Visualization:** Преобразовывать изображения в непрерывные дельта-поля с помощью логарифмической трансформации и выполнять высокоразрешающий пороговый свип (~111k шагов) для обнаружения топологических событий.
2. **Deterministic Knowledge Extraction:** Сжимать массивы весов нейронных сетей (до 111 ГБ) в структурированную карту знаний (~50 МБ) через SVD-декомпозицию и геометрический анализ.
3. **Topological Analysis:** Вычислять персистентные гомологии (Betti-числа, характеристика Эйлера) для анализа связности данных на разных уровнях порогов.
4. **Extensibility:** Обеспечить автоматическое обнаружение новых рендереров и модулей без изменения ядра.

## 4.0 Key Features

### 4.1 Delta Field Pipeline
- **4.1.1** Логарифмическая трансформация пиксельных значений в непрерывное поле D ∈ [-5.546, 5.546]
- **4.1.2** Пороговый свип с настраиваемыми параметрами (sweep-min, sweep-max, sweep-step)
- **4.1.3** Обнаружение jump-событий (изменение occupancy > 1%)
- **4.1.4** 22 визуализации + 1 анимация на источник данных (автоматическое обнаружение рендереров)
- **4.1.5** Параллельное выполнение через ProcessPoolExecutor

### 4.2 Data Sources
- **MNIST** — 10 цифр рукописного ввода
- **Fashion-MNIST** — 10 категорий одежды
- **PNG** — произвольные изображения (латиница, кириллица, портреты)
- **CMYK** — многоканальные TIFF изображения

### 4.3 Nucleus: Deterministic Knowledge System
- **4.3.1** SVD-декомпозиция весов для извлечения детерминированных паттернов
- **4.3.2** Геометрический анализ и построение топологического графа
- **4.3.3** Корреляционный движок для генерации новых паттернов
- **4.3.4** Seed-система для итеративного расширения знаний
- **4.3.5** Universal Knowledge Map — сжатая репрезентация паттернов
- **4.3.6** Geometric Classifier — классификация на основе геометрических инвариантов

### 4.4 Extensibility
- Автоматическое обнаружение модулей `src/renderers/*.py`
- Модульная архитектура nucleus (19+ компонентов)
- Конфигурация через dataclass и environment variables

## 5.0 Non-Functional Requirements
- **Performance:** Поддержка параллельных вычислений (CPU multiprocessing, GPU MPS/CUDA)
- **Memory Efficiency:** Кэширование на уровне сессии, ограничение воркеров до 8
- **Portability:** macOS (MPS) и Linux/Windows (CUDA)
- **Type Safety:** mypy strict mode (за исключением threshold_comparison.py и summary_dashboard.py)
- **Code Quality:** ruff linting + форматирование, line-length = 100
- **Testing:** pytest с тестами для математики и интеграции

## 6.0 Constraints
- Python >= 3.14
- Зависимости: numpy, scipy, matplotlib, Pillow, torch, tqdm, scikit-learn, safetensors, flask
- Данные MNIST/Fashion-MNIST должны быть в формате NPZ
- Документация частично на русском, частично на английском
