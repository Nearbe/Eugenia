# Track: Создать Formatter для приведения Python-кода к inline-формату

## Specification

### 1.0 Цель
Formatter — это не просто форматировщик, а **трансилятор Python-кода**, который приводит любой код к inline-формату с:
- Детерминированными инструментами
- Чётким неймингом в соответствии с назначением инструмента
- Максимальной предсказуемостью

### 2.0 Ключевые принципы
1. **Inline-функции:** Все функции должны быть inline — напрямую встраиваться как список инструментов
2. **Детерминизм:** Один и тот вход → один и тот выход, без вариативности
3. **Чёткий нейминг:** Имена инструментов отражают их точное назначение (per naming convention в product-guidelines.md)
4. **Англицизмов нет:** Нейминг на русском или нейтральный, без кальки

### 3.0 Входные данные
- Python-файлы (`.py`) из `src/`, `tests/`, `conductor/`
- Любой исходный код с произвольным форматированием

### 4.0 Выходные данные
- Приведённый Python-код в inline-формате
- Сохранение функциональности (semantic equivalence)
- Сохранение структуры модулей и пакетов

### 5.0 Требования к инструментам
Каждый инструмент Formatter должен:
- Иметь чёткое, описательное имя
- Быть детерминированным (нет случайности, нет зависимости от окружения)
- Иметь однозначную семантику
