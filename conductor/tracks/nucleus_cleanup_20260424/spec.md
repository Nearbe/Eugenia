### Контекст

- **Eugenia.py и Eugenia_v0.py** — виртуальный контейнер (аналог Docker, но мощнее). Это основная среда исполнения.
- **src/core/** — 32 файла собственной математики проекта (RealMath, ZFC-расширения, p-адические числа, solenoids и т.д.). Чистый Python, без numpy/scipy/torch.
- **src/nucleus/** — 15 Python-файлов, все зависят от numpy (15 файлов), scipy (1), safetensors (1). Модули абсолютно изолированы друг от друга — это главная проблема.
- **generate.py** — мусор, не трогаем.
- **src/orchestrator/** — мусор, забыть.
- **Интеграция с LLM (llama.cpp)** — не входит в трек.
- **Новые визуализации** — позже, не сейчас.

### Ключевые проблемы

1. **src/core/ — возможное переусложнение:** 32 файла, нужно проверить, нет ли излишней абстракции.
2. **Модули nucleus изолированы:** Каждый файл работает автономно, нет единой архитектуры взаимодействия. Это "самая большая проблема и головная боль".

4. **Тесты внутри модулей:** Тесты размещены прямо внутри модулей, а не в `tests/`.

### Протокол взаимодействия

Механический протокол общения между пользователем и AI. Каждый пункт — отдельное правило, нумерация сквозная.

**1. Этап принятия решения** = AI встретился с situation, где нет однозначного ответа в spec/plan/контексте.

**2. Решение принято** = пользователь сказал "да", "утверждаю", "ок", или не ответил "нет"/"изменить" в течение итерации.

**3. Инициатива:** AI иницирует когда: (а) встретил этап принятия решения (п.1), или (б) вариант действия один и определён в spec/plan. Пользователь иницирует в любой момент.

**4. Вопросы:** AI задаёт вопрос только когда: (а) есть 2+ вариантов действия, И (б) spec/plan не определяет приоритет между ними. В остальных случаях — действует.

**5. Без отчётов:** Если пользователь не просил анализ — не отчитываться о проделанной работе. Двойной анализ (AI + пользователь) — только по запросу пользователя.

**6. Фиксация договорённостей:** Каждая договорённость записывается в протокол (этот документ).

**7. Триггеры вопросов (механическое правило):**
```
IF (вариантов действия >= 2) AND (spec/plan не определяет приоритет):
    → ЗАДАТЬ вопрос
ELSE:
    → ВЫБРАТЬ вариант с наименьшим риском
    → ДЕЙСТВОВАТЬ
```

**8. Завершение шага:** Шаг завершён когда выполнены ВСЕ условия:
```
(все поняли) AND (код работает) AND (закоммичено)
```

**9. Конфликты:** Пользователь решает окончательно. AI объясняет последствия решения пользователя.

**10. Нарушение протокола:** Нет механизма наказания. Принцип доверия.

**11. Пользователь всегда главный:** Если пользователь говорит "сделай" — AI делает ровно то, что сказано. Даже если это может сломать. Пользователь знает, что делает.

**12. Изменение протокола:** Любой может предложить изменение в любой момент. Изменение вступает в силу когда оба согласились. Записывается в протокол.

**13. Протокол навсегда:** Применяется ко всему. Меняется только по п.12.

**14. Самопроверка (механическое правило):**
```
ПЕРЕД вопросом к пользователю:
    result = проанализировать_свой_результат()
    IF NOT result.корректен:
        → исправить_результат()
        → НЕ задавать вопрос
    answer = проверить_в_контексте()
    IF answer.найден:
        → действовать без вопроса
    ELSE:
        → задать вопрос
```
