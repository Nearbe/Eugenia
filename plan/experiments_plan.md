## ЧАСТЬ 1: ПЛАН ЭКСПЕРИМЕНТОВ

### Эксперимент 1: Принцип засовывания данных
**Цель:** Понять — помаленьку (stream) или блоками (batch) работает лучше.
**Подход:** Тестовый массив (1GB, 10GB, 50GB) — загрузить обоими способами — измерить время.

### Эксперимент 2: Куда засовывать данные
**Цель:** Понять какой механизм аллокации работает лучше.
**Варианты:** mmap, shared_memory, numpy arrays, ctypes, memoryview.

### Эксперимент 3: Связь с весами модели
**Цель:** Понять как засунуть веса модели в виртуальный контейнер.
**Варианты:** GGUF → mmap, safetensors → mmap, llama.cpp C API, Python-парсинг GGUF.

### Эксперимент 4: GPU эксперименты
**Цель:** Понять что даёт GPU в этом контексте.
**Варианты:** Metal API, Core ML, llama.cpp Metal backend, numpy on GPU.

### Эксперимент 5: IEEE 754 как преимущество
**Цель:** Понять как использовать IEEE 754 ограничения на пользу.
**Варианты:** Inf/NaN как маркеры, переполнение как триггер, denormal numbers, rounding modes.

### Эксперимент 6: Привязка к полу-тактам
**Цель:** Понять как привязать вычисления к полу-тактам процессора.
**Подход:** Изучить setswitchinterval, измерить точность таймингов, связать с тактами.

### Эксперимент 7: Роль Python и GIL
**Цель:** Понять какую роль играет Python в этой архитектуре.
**Вопросы:** GIL = детерминизм = преимущество. Какой overhead? Можно ли заменить на C?
